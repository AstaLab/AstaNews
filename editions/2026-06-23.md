## 1. 🧠 [model] Sakana Fugu Technical Report

Sakana 发布了 Fugu 系列模型，走的是一条和「做一个全能大模型」不太一样的路：把不同厂商各有所长的 LLM 拼成一个更强的整体。Fugu 本身是被训练出来的「编排者」——它读懂你的问题后，临场设计出一套智能体协作方案，决定调哪些模型、怎么组队，从而拿到超过任何单个模型的表现。团队放出两个版本：Fugu 兼顾速度与效果、适合日常，Fugu-Ultra 则在最难的问题上死磕答案质量。

论文称在一串硬基准上对「可公开获取的模型」达到 SOTA，覆盖面很广：软件工程的 SWE-Bench Pro、终端操作的 Terminal Bench、编程的 LiveCodeBench，再加研究生级难题 GPQA-Diamond、超难综合推理 Humanity's Last Exam，以及考图表推理的 CharXiv Reasoning。训练上混用了大规模微调、进化算法和强化学习，进化这条线也是 Sakana 一贯的风格。

要客观看的是：这份技术报告的摘要一个具体分数都没给。所有「SOTA」目前都还是定性说法，没有可比数字、没列对照基线，增益到底多大看不出来。对做多模型协作、router、agent 编排的团队，「把编排当成可训练能力」这个思路值得跟进，但分量得等完整数据和第三方复现才能下定论。

🔗 [一手源](https://arxiv.org/abs/2606.21228)

## 2. 🤖 [agent] Ai2/UW TMax: 开源 RL 终端 agent 配方与 Qwen3.5-9B 终端 agent

Ai2 联合华盛顿大学开源了 TMax：一套训练终端 agent 的强化学习配方，外带一个基于 Qwen3.5 的 9B 终端 agent。对做 coding agent 的团队，这条值得看——它给的不是又一个只能调的闭源 API，而是权重、数据连同训练全过程的 RL rollout 一起摊开，相当于把一条从零做终端 agent 的可复现路径放上了桌面。

所谓终端 agent，就是模型能在命令行里自己敲命令、跑测试、读报错、改文件，一步步把工程任务做完。这类任务多步、长程、每步都有真实副作用，奖励又往往只在最后才出现，靠监督微调学不透，强化学习更合适，但公开的成熟配方一直稀缺。TMax 用 DPPO 算法，在 OpenThoughts 的 TMax-15k 上训练，该数据集含 14600 个可交互的 RL 环境。

成绩上，TMax-9B 在 Terminal Bench 2.0 拿到 27.2%，是 10B 以下最强的开源权重模型，逼近闭源的 Claude Haiku 4.5（29.8%）；在更轻量的 Terminal Bench Lite 上 53.0%，领跑作者自己的各项消融。

客观看：27.2% 说明终端 agent 整体还在早期，离 Claude Haiku 4.5 仍差约 2.6 个百分点，Lite 上的领先也是相对自家消融而言、跨家族对比原文未给。但对想把终端 agent 跑在自己机器上、又想看清 RL 配方每个环节的团队，这是近期最实在的开源参考之一。

🔗 [一手源](https://natolambert.substack.com/p/tmax-an-open-rl-recipe-for-terminal) · [讨论](https://x.com/_akhaliq/status/2069107826329350383)

## 3. 📦 [product] 网易有道开源 Confucius4-TTS：14 语种零参考文本跨语种语音克隆（1.3B，Apache）

网易有道把自家"子曰4.0"体系的语音合成引擎 Confucius4-TTS 开源了。对做数字人、跨境内容、语音产品的团队，这条值得停下来看：它是一个能下载到本地、自己部署、还能商用的国产 TTS 选项，而不是又一个只能调 API 的闭源服务。模型 1.3B 参数，采用宽松的 Apache 协议，开放完整权重，下载 54G 资源包即可离线运行。

它主打的能力是跨语种零样本声音克隆。简单说，你上传一段中文录音，模型就能用同一个音色流利说出日语、英语等共 14 种语言（中英日韩德法西、印尼意泰葡俄马来越南），而且号称不依赖参考文本就能克隆——这是全球首个做到这点的开源 TTS。官方自报 3 秒完成克隆，音色与原声相似度超过 85%，任务准确度 97%，还能从参考音频里提取情感、把语调韵律一起跨语种迁移过去。

要客观看：相似度 85%、准确度 97%、以及"说外语像母语者"这些都是厂商自报口径，怎么测、样本多大原文没说，真实效果得听 demo 或等第三方复测。即便如此，对想摆脱闭源 API、把语音克隆跑在自己机器上的团队，这是近期一个挺实在的国产开源新选项。

🔗 [一手源](https://github.com/netease-youdao/Confucius4-TTS) · [讨论](https://www.leiphone.com/category/industrynews/30qYFuhjh76yBsIV.html)

## 4. 🎛️ [post-training] Keyless Attention: Value-Space Routing and Value-Only Caching

长上下文推理最烧的一笔账，是 KV cache。模型生成时要把每个 token 的 key 和 value 两份缓存都存下来，上下文越长越占显存。这篇 Keyless Attention 的省法很干脆：把 key 这一半整个删掉，注意力只在 query 和 value 上算，缓存里只剩 value 一份，显存和访存开销精确减半 50%，而解码速度还和原来持平甚至更快。

它不只是个工程 trick，还配了套理论解释。作者把注意力看成一个可做不同深度分解的运算：标准注意力是"深度 2"，Keyless 是同一家族里的"深度 m"。在 m=3 时，用一个值空间路由矩阵顶替被删掉的 key 投影，参数量和标准注意力一样多——省的是缓存，不是模型容量。

效果上，在 GPT-2、Pythia、Qwen2、Llama 这 5 个模型、4 种架构里，困惑度有 4 个持平或更优；GPT-2 557M 的下游零样本测试在 5 个常识基准里赢了 4 个，全程保持 50% 缓存缩减。

保留也得说清：最大只验证到 Qwen2 1.5B，下游评测只在一个模型上报，能不能放大到生产级大模型还不知道；摘要也没给逐项数字。但思路够实在——KV cache 这块一直被当成"必须双份"的硬开销，这篇说它可能存在对半压缩的结构空间。

🔗 [一手源](https://arxiv.org/abs/2606.21848)

## 5. 🦾 [embodied] Vesta: A Generalist Embodied Reasoning Model

机器人要在真实环境里干活，得同时会定位、空间推理、导航和长程规划。主流做法是每样能力配一个专门模型，再串成流水线——算力贵，而且前一环出错会沿着链条把错误层层放大。Vesta 换了个思路：把这些能力全收进一个统一的具身基座模型，用一个通才顶替一整队专才。

它靠两样东西撑起来。一是一套规模庞大、专门为培养空间感整理的多模态语料，让模型把语言和物理空间里的实际位置对应起来；二是一个简单的多模态记忆机制，让模型在很长的任务跨度上记得住前面发生了什么，据此往下推理。

结果不算小。多个基准上，Vesta 平均超过各项单任务最强专家模型 20% 以上；更关键的是，它比"每个子任务都挑最强专家"拼成的集成还高 10% 以上——说明它不是只赢在某一项。在真正吃记忆和推理的真实机器人任务上，成功率提升超过 35%。

要客观看：原文只给了相对提升的百分比，没公开具体基准名称、绝对分数、模型参数量和记忆机制的实现细节，是否开源也没提。但"一个通用基座替掉一堆专用模型"这条路，在作者自己的评测里跑通了，对做具身和机器人系统的人是个值得跟的信号。

🔗 [一手源](https://arxiv.org/abs/2606.20905)
