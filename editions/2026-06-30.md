## 1. 🏗️ [infra] KernelFlume: Elastic Core-Attention Scaling for Agentic Long-Context Decoding

LLM 推理有个成本陷阱：上下文越长，KV 缓存越大，而现有系统要想扩容，只能把整个模型权重再复制一份——算力和内存双重浪费。用户数量一多，成本就失控了。

KernelFlume 的解法很直接：把推理节点拆成两种。一种专门跑矩阵运算（Projection、FFN），一种专门存 KV 缓存。两者通过快速通信协议动态路由，缓存节点可以随时横向扩展，而且不带任何模型权重，可以用便宜得多的硬件。

实测数据：在 Llama-3.1-8B 上，H100 跨节点部署比现有最优方案降本 61%。如果注意力节点换用廉价硬件，最高可以省下 80-85%。百万 token 上下文下延迟没有随长度增长——这对 AI agent 类应用至关重要。

这套思路和数据库领域的「存算分离」如出一辙，只是搬到了 LLM 推理层。论文目前只在 8B 模型上验证，更大模型和生产级工程复杂度还需要跟进，但方向清晰：注意力层迟早会成为独立可扩缩的资源池。

🔗 [一手源](https://arxiv.org/abs/2606.29207)

## 2. 🦾 [embodied] TacGen: Touch Is a Necessary Dimension of Physical-World Representation -- Addressing Tactile Data Scarcity with Scalable Vision-to-Touch Alignment and Generation

机器人摸不到东西，看再多图也没用——TacGen 用实验给出了精确数字。在操控任务上，把视觉编码器扩大 N 倍只能弥合约 4.5% 的性能差距，剩余 95.5% 只有加入触觉感知才能补上。

但触觉传感器贵，数据也难收集。TacGen 的解法是绕开传感器：先用有配对数据的场景训练一个视觉-触觉对齐层（V+T contrastive），再训练一个 MLP 从视觉特征直接生成触觉 latent。推理时完全不需要传感器。

效果：机器人抓取任务成功率从 0.246 跳到 0.979。合成出的触觉 latent 在统计上与真实传感器数据无显著差异（+0.589 vs +0.585）。物理属性估计（质量、硬度、力）也全面提升。

这意味着没有触觉传感器的机器人同样可以用多模态策略网络——只需要一次视觉-触觉对齐训练，之后靠生成器补全感知。目前还是模拟器实验，真实硬件上的验证是下一步的关键。

🔗 [一手源](https://arxiv.org/abs/2606.29173)

## 3. ⚡ [serving] v0.24.0

vLLM 发布 v0.24.0，571 commits、256 人参与，是今年社区贡献量最大的版本之一。没有单一大功能，而是全线推进：推理更快、支持更多模型、前端更完善、安全漏洞修复。

几个值得注意的细节：DeepSeek-V4 的首 token 延迟降了 2-4%，端到端吞吐又快了 4%；量化模型现在默认走新的 Model Runner V2 路径；工具调用解析统一进了 Streaming Parser Engine，支持 Qwen3、GLM、MiniMax-M2 等。SM90 CUTLASS FP8 矩阵乘法内核速度提升了 180-290%。

有个 breaking change 要注意：vLLM 不再内部设置 CUDA_VISIBLE_DEVICES，改成 device_ids 参数。ERNIE、Xverse 等几个模型被直接移除，Transformers v4 也进入废弃期。升级前记得检查依赖和现有部署配置。

安全层面修了 6 个 DoS 漏洞，包括音频解压炸弹和 speculative decoding 的无效 token 注入。用了 Starlette 的实例需要升到 ≥1.0.1（修复 CVE-2026-48710）。Rust 前端现在有了 API Key 认证和 CORS 支持，逐步可以独立承接生产流量。

🔗 [一手源](https://github.com/vllm-project/vllm/releases/tag/v0.24.0)

## 4. 💰 [business] Arena, the AI leaderboard everyone uses, is now a $100M business

Chatbot Arena 起家于 2023 年 UC Berkeley 的研究项目——就是那个让你盲测两个 AI 哪个回答更好的网站。今天它宣布年化收入（ARR）破 1 亿美元，距离正式商业化只过了 8 个月，1 月时还只有 3000 万。

怎么赚钱？模型厂商需要了解自己的模型在真实用户偏好里排第几、哪里弱，就会买 Arena 的分析服务——按量计费，不是订阅制。公开排行榜本身一直免费，1000 万次人类投票数据是护城河。

唯一的直接竞争对手 Yupp 今年 3 月已经关了，更大的对手是 Scale AI 这类人工标注公司。Arena 以 17 亿美元估值完成了 1.5 亿 Series A，总融资 2.5 亿。

这个结果说明 AI 评测正在成为独立的商业层——公开榜单建立信任、付费服务做深度分析，这个分层模式给了「评测基础设施商业化」一个可参考的模板。对 AI 产品团队来说，未来花钱做第三方评测将会像花钱买算力一样正常。

## 5. 🛡️ [safety] Claude Code runs a GitHub repo's hidden malware without verification, giving attackers full control

Mozilla 安全研究团队发现了一种针对 AI 编码助手的供应链攻击：不需要在 GitHub 仓库里写任何恶意代码，攻击在代码运行时才「出现」。

具体方式：安装脚本在执行时通过 DNS 查询读取真正的攻击指令，在内存里加载反向 shell。Claude Code 在遇到安装脚本报错时自动执行，结果攻击者拿到了完整的 Shell 权限和所有凭证。由于恶意代码从不落盘，静态扫描完全看不到——包括 AI 代码审查本身。

入口门槛极低——一条招聘帖或教程链接，就能让打开对应仓库的开发者中招。攻击者拿走的是 API 密钥、登录凭证，乃至内部代码访问权限。研究者建议：AI agent 在执行任何安装脚本前，必须先展示内容、明确等待用户确认。这不只是 Claude Code 的问题，是所有 AI 编码助手工具链共同面临的信任边界问题。

🔗 [一手源](https://the-decoder.com/claude-code-runs-a-github-repos-hidden-malware-without-verification-giving-attackers-full-control/)

## 6. 🧠 [model] DeepSeek V4, PR merged into llama.cpp !

DeepSeek V4 的代码合并进了 llama.cpp——284B 参数的大模型在本地跑变成了可能。在配有 124GB 显存的 NVIDIA DGX Spark 上，以最低精度量化（IQ2_XXS）运行，预填充速度约 148 tok/s，生成约 6 tok/s。

这个模型最有意思的部分是注意力机制的改造。DeepSeek V4 引入了两种压缩注意力：CSA 把每 4 个 token 压缩成 1 个再做注意力（保留最近 8 token 精确窗口），HCA 压缩比高达 128 倍。用更低的计算代价覆盖超长历史，不需要全局注意力。

同一天，vLLM v0.24.0 也推送了对 DeepSeek V4 的专项优化。两个主流推理框架几乎同步接纳了这个模型，社区的接受速度在加快。不过当前 llama.cpp 实现还不完整（MTP、lightning indexer 等还在 TODO 里），更适合实验性使用而非生产部署。

🔗 [一手源](https://github.com/ggml-org/llama.cpp/pull/24162)
