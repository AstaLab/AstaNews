# Readiness 改写层：把"信息全"改写成"看得懂"

这一层独立于评分和裁决。前面几步的产物是**信息很全的结构化记录**（每条带 facts、why_matters、links），目的是不丢事实、不搞错；这一层把精选条目改写成**微信群受众**读得下去的稿子。两步分开，是因为"提取信息"和"写得好看"是两种不同的能力，混在一起两边都会打折——也是为什么改写 agent 必须拿到上一步完整的 facts，而不是让它自己去记。

## 受众画像（决定一切）

读者是 Asta Lab 微信群：**全是 AI 从业者，但分布在不同技术栈**。一个做 serving 的人在读具身论文，一个做产品的人在读 post-training 结果。所以：

- 他们**不需要被科普"什么是大模型"**，但**需要一句话桥接到自己不熟的子领域**。写 VLA 论文时，别假设读者知道 VLA 是什么；写 reward model 时，别假设做 infra 的知道 RLHF 细节。
- 他们**看不懂就划走**。第一句话没让他 get 到"这关我什么事"，后面写得再硬核也没用。
- 他们**讨厌营销腔**。"重磅""革命性""王炸"出现即失分。给事实，让读者自己判断分量。

## 三个权衡轴

**易懂 ↔ 技术性**：不是二选一。做法是"先通俗给结论，再补一个技术锚点"。
- ❌ 太通俗没价值："某公司发布了新的注意力机制，效果很好。"
- ❌ 太硬核劝退："MSA 在 GQA 基础上引入 per-group Index Branch 做 block-level top-k KV 选择并与 exp-free kernel 协同。"
- ✅ 平衡："长文本推理最烧的是显存和带宽。MiniMax 把注意力改成'每个查询组只挑最相关的几块 KV 算'（块级稀疏注意力 MSA），并且专门为 GPU 执行路径做了设计——已经在一个 109B 的多模态模型上训练验证过，不是纸面方案。"——通俗结论打头，括号里留技术名词当锚点，硬核的"为硬件执行设计"用大白话说出价值。

**长度**：每条 **3–5 句**，约 80–160 字。一条 digest 不是论文摘要。信息密度靠"砍掉读者不需要的铺垫"来提，不是靠塞满术语。量化点挑**最有冲击力的 1–2 个**编进正文，其余留给网页的 facts 列表。

**忠实 ↔ 流畅**：改写不能改事实。facts 里的数字、版本号、是否开源/有无权重，一个都不能错或丢。拿不准的、原文没明说的，宁可不写，绝不脑补（"据称""官方未给出具体数字"是诚实的写法）。自报基准要标"自报/in-house"，别当客观结论。

## 每条的结构（不是模板，是顺序）

1. **第一句 = 关你什么事**：用读者熟悉的语言说清"发生了什么、为什么值得停下来"。跨栈桥接句通常在这。
2. **中间 = 硬核锚点**：一个关键技术名词 + 一句话解释它，或一个最有分量的量化对比。
3. **收尾 = 落到判断**：它意味着什么、对谁有用、有什么保留（落后闭源前沿？只是 tag 没正式 notes？自报基准？）。诚实的保留比吹捧更可信。

## 整期的开场

`overview`（一句话）和 `headline`（标题）也由这一层写。overview 抓**当天的主线**——今天多条新闻背后共同的故事是什么（"开源阵营在 coding 模型上集体冲刺"远好于"今天有几条模型新闻"）。没有主线就老实说最大的一条。

## 示例改写

**输入记录**（信息全但生硬）：
```
layer: model, source: hf-trending-models
title: Moonshot 开源 Kimi-K2.7-Code
facts: ["1T 总参/32B 激活 MoE", "256K 上下文", "原生 INT4", "modified-MIT 权重",
        "KCB v2 自报 62.0（K2.6 50.9, GPT-5.5 69.0, Opus 4.8 67.4）", "day-0 vLLM/SGLang"]
why_matters: 目前能本地部署的最强 coding 权重之一
```

**输出 readable**（微信版）：
> Moonshot 把自家旗舰 coding 模型开了源——这是现在你能下载到本地、自己部署的最强编程模型之一。规格很顶：1T 总参数但每次只激活 32B（MoE 架构，省算力），256K 上下文，原生 INT4 量化，权重用接近 MIT 的宽松协议，vLLM/SGLang 第一天就能跑。自报编程基准 62 分（上一代 50.9），但仍落后 GPT-5.5 和 Opus——而且基准是自家的，参考着看。对想摆脱闭源 API、自建 coding agent 的团队，这是个实打实的新选项。

注意：跨栈桥接（"能下载到本地自己部署"）打头，MoE 用"每次只激活 32B，省算力"一句解释，自报基准明确标注并给出闭源对照，收尾落到"对谁有用"。facts 里的数字一个没丢，但读起来是给人看的。
