# Readiness 改写层：把"信息全"改写成"看得懂"

这一层独立于评分和裁决。前面几步的产物是**信息很全的结构化记录**（每条带 facts、why_matters、links），目的是不丢事实、不搞错；这一层把精选条目改写成**微信群受众**读得下去的稿子。两步分开，是因为"提取信息"和"写得好看"是两种不同的能力，混在一起两边都会打折——也是为什么改写 agent 必须拿到上一步完整的 facts，而不是让它自己去记。

## 受众画像（决定一切）

读者是 Asta Lab 微信群：**全是 AI 从业者，但分布在不同技术栈**。一个做 serving 的人在读具身论文，一个做产品的人在读 post-training 结果。所以：

- 他们**不需要被科普"什么是大模型"**，但**需要一句话桥接到自己不熟的子领域**。写 VLA 论文时，别假设读者知道 VLA 是什么；写 reward model 时，别假设做 infra 的知道 RLHF 细节。
- 他们**看不懂就划走**。第一句话没让他 get 到"这关我什么事"，后面写得再硬核也没用。
- 他们**讨厌营销腔**。"重磅""革命性""王炸"出现即失分。给事实，让读者自己判断分量。

## 三个权衡轴

**易懂 ↔ 技术性**：不是二选一。做法是"先通俗给结论，再补一个技术锚点"。
- ❌ 太通俗没价值："某公司发布了新的注意力机制，效果很好。"
- ❌ 太硬核劝退："MSA 在 GQA 基础上引入 per-group Index Branch 做 block-level top-k KV 选择并与 exp-free kernel 协同。"
- ✅ 平衡："长文本推理最烧的是显存和带宽。MiniMax 把注意力改成'每个查询组只挑最相关的几块 KV 算'（块级稀疏注意力 MSA），并且专门为 GPU 执行路径做了设计——已经在一个 109B 的多模态模型上训练验证过，不是纸面方案。"——通俗结论打头，括号里留技术名词当锚点，硬核的"为硬件执行设计"用大白话说出价值。

**长度**：**新闻体，3–4 段，约 250–420 字**。给信息足够的呼吸空间——一段说清"发生了什么、为什么停下来看"，一段展开技术内核和关键数字，一段给背景/对比/它在格局里的位置，一段落到判断与保留。不是论文摘要的密集堆砌，也不是一句话带过；像一篇写给同行看的科技新闻短稿。段落之间用空行分隔。量化点可以多带几个（编进叙述里，别堆成清单），最硬核的细节仍留给网页的 facts 列表。宁可多写一句把术语讲透，也不要为了短而留下读者看不懂的坑。

**忠实 ↔ 流畅**：改写不能改事实。facts 里的数字、版本号、是否开源/有无权重，一个都不能错或丢。拿不准的、原文没明说的，宁可不写，绝不脑补（"据称""官方未给出具体数字"是诚实的写法）。自报基准要标"自报/in-house"，别当客观结论。

## 段落顺序（不是模板，是新闻体的自然展开）

1. **导语段 = 关你什么事**：用读者熟悉的语言说清"发生了什么、为什么值得停下来"。跨栈桥接句通常在这——给不熟这个子领域的人一个抓手。
2. **展开段 = 技术内核**：把关键技术名词讲透（名词 + 一句话解释它怎么工作/解决了什么），带上最有分量的几个量化数字，融进叙述而不是堆清单。需要的话可以拆成两段：一段讲方法，一段讲结果。
3. **背景/对比段**：它相对此前/同类是什么位置，谁在做类似的事，为什么是现在。给读者一个坐标系。
4. **落点段 = 判断与保留**：它意味着什么、对谁有用，以及诚实的保留（落后闭源前沿？只是 tag 没正式 notes？自报基准？样本量小？）。诚实的保留比吹捧更可信。

不是每条都要硬凑满四段——有的事实简单，三段足矣；复杂的可以四段。关键是读起来像一篇完整的短新闻，而不是一句话或一份清单。

## 网站两层正文：摘要（列表）+ 深读（详情页）

网站每条新闻有两层内容，**不是同一段**（用户铁律：列表里看到的是摘要、点进详情页必须更深更详细——不能进去还是那段摘要）：

- **`summary`（列表卡片）**：**一段完整的话，约 150–260 字 / 3–5 句**（不是一两句带过、要成段；但也别写成长文）。自成一体讲清「发生了什么 + 最关键的点/数字 + 为什么值得看」。**单段、不要换行**——单条新闻的摘要不分行（前端列表按整段渲染）。**所有 group/daily 条目都要有**。
- **`deep`（详情页 `/item/` 正文）**：**深度全文解读，约 600–1000 字 / 5–8 段**，明显比 `summary` 和微信版 `readable` 更深更详细。点进来的人是想真正搞懂这件事——别让他只看到摘要。结构（新闻体自然展开，不是填模板）：导语桥接「关你什么事」→ 背景与来龙去脉 → 技术内核/方法逐项讲透（术语就地解释）→ 关键数字与对比坐标 → 影响：对哪类从业者意味着什么、怎么用 → 诚实的局限与存疑。facts 的数字一个不能错或丢，拿不准的不脑补。**所有 group/daily 条目都要有**。

`readable`（微信新闻体 3–4 段，concise）**仅 group 精选需要**——微信群发用它，保持 30 秒读完的节奏。daily 非精选不进微信，无需 `readable`。

（旧字段 `take`、`sharp`（锐评）、`source_excerpt` 都不再产出——产品求简，每条只保留 `summary`+`deep`，group 另加微信 `readable`。）

## 整期的开场

整期由这一层写 `overview`（当期摘要）与 `headline`（短标题）。

**站点的呈现：每期的大标题就是日期**（如「2026-06-15 · 周一」，weekday 用「周X」短写法）。`overview` 紧跟日期下、作为**当期摘要(abstract)单独成段**；`headline` 只用于归档列表的一行副标题与浏览器/SEO 标题。所以分别这样写：

- **overview = 当期摘要（日报摘要）**：新闻体、约 200–380 字，要**自成一体**。开门见山点出当天主线（今天多条新闻背后共同的故事是什么，「开源阵营在 coding 模型上集体冲刺」远好于「今天有几条模型新闻」），再把几条重点串起来。没有主线就老实说最大的一条。它前面没有别的标题替它起头，别写得像接在某句话后面。
  - **必须换行分段**：因为它汇总当天很多条新闻，要用换行把不同新闻簇分开——一句总起占一行，之后每个主题/板块（如「安全侧…」「评测侧…」「系统与产品侧…」「具身长尾…」）各占一行。在 `overview` 字符串里用真实换行符 `\n` 分隔（前端 `.ed-abstract` 用 `white-space: pre-line` 渲染）。**这是「日报摘要」专属；单条新闻的 `summary` 不分行。**
- **headline = 一行短标题**：**≤24 字、两个对称分句**（学「前沿实验室扎堆冲 IPO，技术面多点开花」），不堆三段以上。和 overview 各写各的，可呼应但 overview 不必照抄 headline 的措辞。

⚠️ **两者都是写给读者的，不是写给我们自己的工作记录。** 只讲新闻本身，绝不暴露策展过程：不要写"这期 N 条""严格落在 X 月 X 日""媒体转载的旧论文一律剔除""周末没有发布所以…""本次窗口"这类话——窗口、条数、去重/剔除规则、新鲜度判定全是我们的内部 QA，读者看到只会觉得在读注释。直接像报纸导语那样开门见山。headline 同理：是新闻标题，不是流程说明。

## 示例改写

**输入记录**（信息全但生硬）：
```
layer: model, source: hf-trending-models
title: Moonshot 开源 Kimi-K2.7-Code
facts: ["1T 总参/32B 激活 MoE", "256K 上下文", "原生 INT4", "modified-MIT 权重",
        "KCB v2 自报 62.0（K2.6 50.9, GPT-5.5 69.0, Opus 4.8 67.4）", "day-0 vLLM/SGLang"]
why_matters: 目前能本地部署的最强 coding 权重之一
```

**输出 readable**（微信版，新闻体多段）：
> Moonshot 今天把自家旗舰编程模型 Kimi-K2.7-Code 开了源。对做 coding agent 的团队，这条值得停下来看：它是目前你能直接下载到本地、自己部署的最强编程模型之一，而不是又一个只能调 API 的闭源模型。
>
> 规格相当顶。模型总参数 1T，但用了 MoE 架构——每次推理只激活其中 32B，算力开销远小于参数规模看上去的样子；上下文 256K，原生 INT4 量化，还内置了一个 400M 参数的视觉编码器。权重采用接近 MIT 的宽松协议，vLLM、SGLang、KTransformers 第一天就支持部署。官方自报的编程基准 KCB v2 拿到 62 分，比上一代 K2.6 的 50.9 有明显跳升，thinking token 用量还降了约 30%。
>
> 不过要客观看：62 分仍落后 GPT-5.5 的 69 和 Opus 4.8 的 67.4，而且这套基准是 Moonshot 自建的、属于自报成绩，真实差距得等第三方未污染评测。即便如此，对想摆脱闭源 API 依赖、把 coding agent 跑在自己机器上的团队，这是近期最实在的一个新选项。

注意四段的展开：导语（跨栈桥接"能本地部署"）→ 技术展开（MoE/INT4/基准，数字融进叙述不堆清单）→ 保留+落点（自报基准、落后闭源，但对谁有用）。术语就地讲透（MoE = 每次只激活 32B），facts 里的数字一个没丢，但读起来是一篇给同行看的短新闻。
