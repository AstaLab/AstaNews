## 1. 🛡️ [safety] Claude Fable 5 / Mythos 5 系统卡公开：新一代「最强公开模型」的安全画像摊上桌面

新模型上线只是第一步，真正能让做安全和对齐的人停下来细读的，是随之公开的系统卡（System Card）。这天 Anthropic 为前一天首发的 Claude Fable 5 与 Mythos 5 放出了系统卡，长期写 AI 安全评注的 Zvi Mowshowitz 把它逐条拆开来读。

所谓系统卡，是模型方在发布时披露能力评估、红队测试和已知风险的官方文档——它不讲营销话术，而是回答「这个模型能干什么、在哪些方向上危险、我们测了什么」。对一个被称作当前最强的公开模型来说，这份卡就是外界审视它的第一手依据。

Zvi 的判断没有绕弯子：在他看来，Claude Fable 5 是目前公开可用的最强模型。至于具体强在哪、安全测试覆盖到什么程度，他的长文是对一手系统卡的逐条评注，量化细节仍以官方文档为准——这也是为什么对从业者而言，原始系统卡比任何转述都值得亲自读一遍。

把这件事放进更大的背景：当一个实验室的旗舰模型强到被当成行业标尺，它的发布就不再只是产品事件，而是安全披露事件。系统卡写得多透明、外界读得多仔细，正在成为衡量前沿发布是否「负责任」的一把尺子。

## 3. 🧠 [model] Google 放出 Gemini-SQL2：Gemini 3.1 Pro 文本转 SQL 在 BIRD 单模型榜拿到 80.04%

对任何想让非技术同事「用大白话查数据库」的团队，文本转 SQL（text-to-SQL）的准确率每往上挪几个点，落地的可能性就实一分。这天 Google Research 放出了 Gemini-SQL2，底座是 Gemini 3.1 Pro。

它最硬的数字是：在 BIRD 单模型排行榜上拿到 80.04% 的执行准确率。这里要把术语讲透——BIRD 是专门衡量「自然语言 → SQL」在真实、复杂数据库上表现的基准，比早期那种玩具数据集难得多；而「执行准确率」（execution accuracy）不是看 SQL 长得像不像，而是把生成的 SQL 真跑一遍、看查出来的结果对不对。80% 出头的执行准确率，意味着大多数自然语言提问能被翻成跑得通、答得对的查询。

Gemini-SQL2 配套给出的是一套 schema 接地（schema-grounded）的实现范式——简单说，就是把数据库表结构喂给模型当上下文，让它的生成被真实字段约束住，少出「查一个不存在的列」这类错误。

要客观看：MarkTechPost 也点出 Google 还有部分实现细节没有公开，单模型榜的成绩与真实生产环境中千奇百怪的库、脏数据、模糊问法之间，仍隔着工程距离。但对正在搭数据问答、BI copilot 的人，80.04% 这个公开可比的数字，至少把「值不值得现在就上」的判断往前推了一步。

🔗 [一手源](https://www.marktechpost.com/2026/06/12/google-releases-gemini-sql2-gemini-3-1-pro-text-to-sql-scores-80-04-on-bird-single-model-leaderboard/)

## 5. 🤖 [agent] Moonshot 推出 Kimi Work：能在本地桌面跑 300 个子 agent 的「agent 蜂群」

做 agent 的人这两年都在追同一个问题：怎么让模型不只在对话框里聊天，而是真去操作你的电脑、浏览器、文件，把活儿干完。Moonshot 这天给出的答案叫 Kimi Work——一款直接装在 macOS 和 Windows 上的本地桌面 agent。

它最有辨识度的设计是「蜂群」（swarm）：不是一个 agent 单线程地慢慢做，而是同时调度多达 300 个子 agent 并行处理任务。配合它通过 WebBridge 直接驱动你已经登录的浏览器（也就是借用你现成的登录态去网站上操作）、并能排布后台定时任务，整套东西想做的是「把一个能自己跑腿的数字助手装进桌面」。

据 MarkTechPost 报道，Kimi Work 底层跑的是 Kimi K2.6，不过这一点是「据称」（reportedly），Moonshot 在该报道里并未完全确认模型版本与部署细节。

把它放进格局看：从云端 agent 到本地桌面 agent，竞争正从「模型多聪明」转向「能不能安全、稳定地接管真实环境」。300 个子 agent 的蜂群、直接操控已登录浏览器，这些设计在效率诱人的同时，也把权限与安全的口子开得更大——这正是做桌面 agent 绕不过去的一体两面。对在搭 agent 产品的团队，Kimi Work 值得当成一个现实参照：消费级的「能干活的 agent」正在成形。

🔗 [一手源](https://www.marktechpost.com/2026/06/12/moonshot-ai-launches-kimi-work-a-local-desktop-agent-reportedly-running-on-kimi-k2-6-with-a-300-sub-agent-agent-swarm/)

## 6. ⚡ [serving] sglang v0.5.13：树状投机解码定为默认路径，Day-0 接住 Nemotron 3 Ultra

对做模型 serving 的团队，sglang 的版本号往往比模型发布更值得盯——它决定你手里的卡能跑多快、稳不稳。这天的 v0.5.13 是一次有实打实分量的更新，核心是把投机解码（speculative decoding）这条加速路径彻底换代。

所谓投机解码，是用一个轻量「起草」过程先猜出一串候选 token、再交主模型一次性验证，从而少走几趟昂贵的前向计算来提速。v0.5.13 把 Spec V2 定为默认：支持 topk>1 的树状起草（tree drafting，一次猜出多条候选分支再验证），并在 triton、FA3、MLA、aiter 各后端都做到生产可用，连 page_size>1 和 Mamba/混合线性模型也覆盖到了。旧的 Spec V1 被弃用，EAGLE、MTP 统一迁到 V2 worker。

模型支持走「Day-0」策略：Nemotron 3 Ultra 发布即可跑（附官方博客），同时新增 Step-3.7-Flash、Command A+ 等自回归模型与 Cosmos3、FLUX.2-Klein 等扩散模型。底层还顺手抠了调度开销——通过 FutureMap 统一异步传值、把 prefill 输入传输挪到 forward 流，降低每步启动开销、提升高并发稳定性，Piecewise / Breakable CUDA Graph 的覆盖也扩展到 DSA、Kimi-K2.5、DeepSeek V4。

对在生产上扛高并发的人，这一版不是纸面 changelog，而是能直接换算成吞吐与延迟收益的更新。
