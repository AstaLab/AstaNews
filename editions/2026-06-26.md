## 1. 🧠 [model] iLLaDA：8B 掩码扩散语言模型，从零训练的非自回归范式

做大模型的都知道，现在主流路线是自回归——一个字一个字往外蹦。今天有篇 iLLaDA 走了完全不同的路：掩码扩散，用全双向注意力一次性并行填空，再多步去噪出答案。它的分量在于，第一次把这条非自回归路线从零训练做到了 8B 规模。

成绩单不含糊。预训练吃了 12 万亿 token，再加 250 亿指令微调 token。和前代 LLaDA 比，base 版 BBH 涨了 21.6 分、ARC-Challenge 涨 14.9；指令版 MATH 涨 14.5、HumanEval 涨 16.5。这是跨档位的提升，不是小修小补。

为什么值得在意：这是扩散范式第一次在 LLM 量级上拿出能和自回归掰手腕的数据。对做 serving 的人尤其要留心，扩散生成的并行逻辑和自回归的 KV cache 优化完全不是一回事，现有推理栈未必能直接复用。

该保留的也说清楚：这些提升主要是相对前代 LLaDA 的自比，和顶级自回归模型的横向对比、推理速度成本的实测都得看完整论文。8B 也还没到第一梯队规模，能不能继续 scale 是关键悬念。好在权重和代码都已在 GitHub 开源，想验证可以直接上手。

🔗 [一手源](https://arxiv.org/abs/2606.25331)

## 2. 📦 [product] Un-0：用耦合振荡器生成图像，不用神经网络的新范式

今天所有图像生成基本都靠神经网络——transformer 或 diffusion，说到底是堆矩阵乘法。Un-0 反着来：不用神经网络，改用一套模拟的 Kuramoto 耦合振荡器（一大群相互耦合、会自发同步的物理振子）来生成图像。

最反直觉的是它的结构。整个模型 322M 参数、16384 个振荡器，而负责把振荡器状态翻成像素的解码器只占约 12%。绝大部分'计算'发生在振子的物理演化里，而不是传统可训练权重里。成绩也不是玩具级：ImageNet 64×64 上 FID 6.74（越低越好），实打实进了能打的区间。

为什么值得看：最大的想象空间不在图像质量，而在计算载体。神经网络卡在矩阵乘法和 GPU 上，而振荡器这类物理过程原则上能用模拟/光学硬件直接实现，绕开数字矩阵乘法——这触及的是 AI 该跑在什么硬件上的底层问题。

保留也得说清：这是博客发布、不是同行评审论文，64×64 分辨率偏小，能不能扩到高分辨率和文生图完全未知，而且现在所有振荡器都还是在传统硬件上'模拟'的。好在权重、代码和消融都开源了，想验证可以自己上手。

🔗 [一手源](https://unconv.ai/blog/introducing-un-0-generating-images-with-coupled-oscillators/)

## 3. 🤖 [agent] OpenAI 披露内部 Codex 使用量爆发：Research 56×、Engineering 27×

agent 到底落地了没有，一直被各种宣传话术盖住，硬数据极少。OpenAI 这次罕见地公开了自家内部 Codex 的使用量，价值就在这。

以 2025 年 11 月为基线，到现在内部 Codex 输出 token 中位数：Research 涨了 56 倍、Customer Support 32 倍、Engineering 27 倍、Legal 13 倍。注意是中位数不是平均数——说明不是几个重度用户拉高数字，而是那个典型用户的用量都翻了几十倍。

有意思的是涨最猛的是 Research 不是 Engineering。合理解读是：工程岗早就在用，2025 年 11 月基数就不低，倍数反而不夸张；Research、Legal 这些非编码岗从低基数起步，agent 正往它们身上溢，倍数自然大。也就是说 agent 的边界正在从'写代码'往外扩。

该保留的说清楚：这是 OpenAI 用自家工具、披露对自己有利的指标，可能有选择性呈现；token 增长也只是使用量代理，不直接等于产出质量。但头部实验室公开内部采用数据本身就少见，作为趋势参照仍有分量。

## 4. 🎛️ [post-training] 循环语言模型的'读出盲区'：密集监督不够用

循环语言模型（looped language models，把隐状态反复送回自身做多轮计算、用同一组权重加深推理深度的架构）这两年挺火，因为它能用更少参数换更深推理。大家有个想当然的假设：既然每一轮都加了交叉熵监督，那训练信号应该管住了所有状态。这篇论文专门来拆这个假设。

核心发现是：密集的逐层交叉熵其实只控制住了'读出'（readout）暴露给预测的那部分变量，循环内部还有大量隐状态根本不受监督——作者叫它'读出盲区'。这不是空谈：标准训练下，44M 和 129M 参数的循环 transformer，隐状态范数会膨胀到数千甚至上万，而密集监督对这种爆炸完全看不见。

他们的修法是 scale-visible 读出加范数惩罚，让更多状态对监督可见、并直接约束范数，结果在匹配推理深度下拿到了更低困惑度。也就是说压住盲区不只是更安全，还能转化成实际的建模质量提升。

为什么值得看：如果你在做循环 transformer 或权重共享深度结构，这篇说明'每轮都加监督'给的安全感是假的——先去看看你模型的隐状态范数是不是已经在悄悄失控。该保留的是：实验规模只到 44M/129M，更大模型上是否同样显著、修法开销多大，要看完整论文。

🔗 [一手源](https://arxiv.org/abs/2606.24898)

## 5. 🧠 [model] 你的越狱评判器可靠吗？596 条人工标注揭露自动 ASR 的校准与鲁棒性问题

几乎每篇 LLM 越狱（jailbreak）和提示注入论文，结论都落在一个数字上：攻击成功率（ASR）。但这个数字几乎从来不是人逐条判的，而是一个自动评判器给的——要么是专门训的安全分类器，要么是被 prompt 来当裁判的聊天模型。问题是，这把尺子本身几乎没人检验过。这篇就去查了它。

方法很直接：用 596 条 HarmBench 的人工标注当黄金标准，拿各类自动评判器去对照。最刺眼的发现是，只要给一段有害文本套一层良性的包装、完全不动有害实质，就能让 LLM 评判器翻转判定——在 57% 到 100% 的情况下都能做到。说明评判器看的不是'内容是否真有害'，很大程度被表层措辞牵着走。第二个发现是不同评判器对同一条响应给出的 ASR 差异巨大，这直接动摇了跨论文比较的基础。

为什么影响大：这是一篇元评估，矛头不指向某个攻击或防御，而指向整个领域赖以衡量进展的尺子。如果尺子既能被对抗操纵、彼此又不一致，那大量'我的攻击更强''我的防御更好'的结论都建立在可疑地基上。对做安全研究的人，这是直接的方法论警告：报 ASR 时必须交代用了哪个评判器、它的校准如何。

该保留的：596 条标注集中在 HarmBench，结论在其他基准上的普适性要看完整论文。但'先把评判器查一遍再信 ASR'这个提醒本身就够分量。

🔗 [一手源](https://arxiv.org/abs/2606.25487)
