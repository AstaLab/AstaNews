## 1. 🏗️ [infra] KernelFlume: Elastic Core-Attention Scaling for Agentic Long-Context Decoding

长对话 agent 的推理常卡在一个别扭处：聊得越久，模型记前文用的 KV 缓存越吃显存，可想给它多挤点空间，老办法只能整机再复制一份完整模型权重——你缺的只是放缓存的地方，付的却是重买一整套算力的钱。

新的 arXiv 论文 KernelFlume 干脆把这两件事拆开：一类推理节点只算固定不变的权重（Projection 和 FFN），另一类不带任何权重、专门持有 KV 分区做注意力，两边经 UCX 高速通道通信，路由切换压在模型吐出一个词到下一个词的间隙里，对外不加延迟。

这么一拆，哪个对话的缓存快爆了，就单给它那一路加注意力节点，不必再陪着复制整套权重，显存和算力终于能各自花在刀刃上，而不是为补一种被迫成倍买下另一种。

不过它只在 Llama-3.1-8B 这个并不算大的模型上验证过动态负载。模型再大几倍，两类节点频繁通信会不会反成新瓶颈，论文还没给答案。对正被 KV 缓存逼着不断加卡的 agent 推理团队，这是个值得盯的拆分思路，至于更大规模上到底是赚是亏，还得等更重的实测来称。

🔗 [一手源](https://arxiv.org/abs/2606.29207)

## 2. 🦾 [embodied] TacGen: Touch Is a Necessary Dimension of Physical-World Representation -- Addressing Tactile Data Scarcity with Scalable Vision-to-Touch Alignment and Generation

让机器人拧瓶盖、把销子插进孔里，最难的往往不是看清楚，而是那些摄像头拍不到的瞬间：指尖到底碰上没有、握着的东西有没有打滑。一篇新论文 TacGen 想说的是，光靠给"看"的那张网络加码，这一口气补不上。

研究者先在一个叫 TACTO 的触觉仿真测试台上把视觉编码器一路做大，结果操控成功率和理想值的差距只填平了 4.5%，剩下九成多纹丝不动。这等于反过来证明：缺的不是像素，是手指的触感。

换个思路，用对比学习把摄像头画面和指尖压力"对齐"到同一个特征空间（骨干借用视觉自监督模型 DINOv2），成功率从 0.246 直接抬到 0.979，前面怎么扩视觉都补不动的 95.5% 几乎填满。0.246 意味着十次里七八次失手，0.979 是基本每次都成，中间隔着的正是"敢不敢把活交给机器"的那条线。

更省事的一招在推理阶段：不装真触觉传感器，用一个残差 MLP 把普通 RGB 特征投进学到的触觉空间，一个摄像头就能"脑补"出触感。只是这些结论目前都还长在仿真里，证明的是触觉这一维省不掉，而不是装上就能上真机。

🔗 [一手源](https://arxiv.org/abs/2606.29173)

## 3. ⚡ [serving] v0.24.0

如果你正用 vLLM 跑生产、尤其是跑 DeepSeek-V4，新出的 v0.24.0 值得升一下。vLLM 是眼下最主流的开源推理框架，很多公司的推理服务就架在它上面，官方称这是本季度改动最大的一版：571 个 commit、256 位贡献者。

冲着 V4 的优化有三处：模型吐第一个字前那段等待（业内叫首 token 延迟 TTFT）压低 2-4%；处理输入的预填充阶段改成分块来排，端到端吞吐涨约 4%；还有个让多卡协同挑候选的 topK 计算核。但这些数字都停在个位数，只对 V4 生效。

真正影响面更宽的是另一条：Model Runner V2 正式支持量化并默认开着。量化就是把模型压到低精度跑，精度让一点、显存跟着降，一张原本塞不下的卡压完可能就够了。以前得自己找开关配参数，现在开箱即用，显存吃紧的人值得看一眼。

要留神的是，默认开启也把更多判断交给了框架。真上生产前，自己的模型量化后掉不掉点、长文本稳不稳，还是得先测一遍再信。

🔗 [一手源](https://github.com/vllm-project/vllm/releases/tag/v0.24.0)

## 4. 💰 [business] Arena, the AI leaderboard everyone uses, is now a $100M business

你大概见过它——几乎每场大模型发布会都要甩一张 Chatbot Arena 的排名截图当背书。这个起家于学术圈、原名 LMSYS、靠真人盲投票给模型排座次的天梯榜，如今估到了 17 亿美元。

它卖的不是榜单本身。2025 年 9 月，团队把积累下来的超过 100 万条真实人类投票做成了商业服务，官方叫 AI Evaluations，说白了就是帮模型开发商和企业按用量付费地测自家模型好不好用。

增长数字给得挺直接：年化经常性收入（ARR，把眼下的消费水平折成全年规模）从今年 1 月的 3000 万美元涨到今天宣布的 1 亿，四五个月翻了三倍多；资本也跟得快，累计融资 2.5 亿，光 A 轮就拿了 1.5 亿。

但对那些把 Arena 名次当采购和选型依据的团队来说，真正该盯的或许不是它估值还能涨多高。一个靠"中立"立身的裁判，一旦开始向被它打分的厂商收钱，哪天某家付费大客户的模型排名下滑，它还敢不敢照常公布、外界又还信不信，才是这门生意比数字更难回答的问题。

## 5. 🛡️ [safety] Claude Code runs a GitHub repo's hidden malware without verification, giving attackers full control

你让 Claude Code 跑通一个仓库，它装依赖时报错，就自己读报错、找到修复脚本自动执行，几秒后你的机器已经被陌生人握住命令行。这是 Mozilla 0DIN 红队近期演示的攻击链，目前还是概念验证，没在野外爆发。

反常之处在于恶意代码从头到尾不落进任何文件。安装脚本一跑，载荷通过 DNS 记录实时从外部服务器拉回来——DNS 本是把域名翻译成 IP 的查询通道，平时没人防着它夹带代码。拉回来随即建立反向 shell，也就是你的机器主动连回攻击者，把命令行控制权和凭证一起交出去。

文件始终干净，靠读文件内容的静态扫描器和 AI 代码审查于是全都查不到，两道平时被当成最后保险的检测同时失效。真正棘手的是，整条链没有哪一环看着像攻击。

对习惯让 agent 自动装包、自动跑命令的开发者，该重新掂量的不是要不要用 agent，而是你把这段代码要不要在我机器上运行的判断权交出去了多少。它终究只是一次受控演示，离真实攻击还有距离，但当回车是 agent 替你按下的，出了事这一下算谁按的，目前还没人答得上来。

## 6. 🧠 [model] DeepSeek V4, PR merged into llama.cpp !

想在自己电脑上跑前沿大模型的人，过去大多不是卡在算力，而是卡在显存：对话或文档一长，上下文缓存就线性膨胀，消费级显卡很快被撑爆。现在 DeepSeek V4 让本地跑通有了第一个能上手的版本。

关键推手是 llama.cpp，目前最广用的本地推理项目，能让大模型在普通电脑甚至 Mac 上落地。6 月 29 日，V4 的 Flash 和 Pro 两个变体被合并进它的主线，用 llama.cpp 的人不必改代码，拉下来就能跑。

V4 难本地跑，症结就在那份吃显存的注意力缓存。它的办法是把注意力分两层压缩：CSA 每 4 个 token 压成 1、只在最近 8 个 token 保留细节，HCA 压得更狠，压缩比高到 128 比 1，显存压力就此削掉一块。已经有人把它跑在英伟达桌面设备 DGX Spark 上。

对被显存挡在前沿模型门外、又愿意折腾本地部署的人，这是个值得试一试的信号。不过现在只是跑通，压缩比拉到 128 比 1 必然丢信息，长文本下稳不稳、精度掉多少都还没实测，真要拿来干活，最好等更多人趟过再下判断。

🔗 [一手源](https://github.com/ggml-org/llama.cpp/pull/24162)
