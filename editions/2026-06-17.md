## 1. 🦾 [embodied] 阿里发千问首个具身大模型 Qwen-Robot 系列：操作 / 导航 / 世界模型三连发

6 月 16 日，阿里千问一口气放出三个模型，凑成了 Qwen 家族第一个完整的具身智能系列 Qwen-Robot——给机器人配齐了「会动的手、认路的脚、会想的脑」：负责操作的 VLA 模型 RobotManip、负责移动的 VLN 模型 RobotNav，以及作为「世界模型」的 RobotWorld。三者既能单独部署，也能协同跑，目标是给不同形态的机器人一套通用底座。

数字是这次的硬通货。RobotManip 建在 Qwen3.5-4B 上，吃了约 3.8 万小时操作数据，在 RoboChallenge Table30-v1 上拿下第一(相对提升约 20%)、LIBERO-Plus 到 91.4%。RobotWorld 用冻结的 Qwen2.5-VL 编码器接一个 60 层双流 MMDiT(约 200 亿参数)，靠 860 万视频-文本对、两亿多帧训练，在 EWMBench 和 DreamGen Bench 两个世界模型基准上都排第一。RobotNav 则覆盖 2B/4B/8B 三档，VLN-CE RxR 成功率 76.5%、HM3Dv2 物体导航 75.6%。

对做具身的团队，这条的意义是「大厂把操作 + 导航 + 世界模型打包成一套带基准的开放底座」——RobotManip 和 RobotNav 已开源代码，可以直接拿来对照自己的方案；RobotWorld 目前还只有论文，想复现得等权重。要留意的是「开放」是部分开放，不是三件套全给。

## 2. 🤖 [agent] CmdNeedle：测了 1709 份真实命令黑名单，近七成到九成可被绕过，连 Claude Code 也漏

现在很多终端类 AI agent(在命令行里帮你跑命令的那种)靠一份「命令黑名单」来防止它干危险的事——把 rm -rf、curl 管道执行之类的命令拉黑。CmdNeedle 这篇 6 月 16 日的论文，系统地测了这套做法到底有多靠谱：从 GitHub 上扒了 1709 份真实在用的黑名单、合计 13332 条规则，逐一找绕过方法。

结果不乐观——其中 69.0% 到 98.6% 的黑名单都是「脆弱的」，也就是存在能被绕过的命令；连 Claude Code 那份维护得相当好的内置黑名单，也漏掉了可绕过的命令。原因不难理解：shell 的表达能力太强，同一个危险操作可以换无数种写法，靠枚举「禁止哪些」永远列不全。

对正在给编程 / 运维 agent 加「命令护栏」的团队，这条是一记提醒：denylist 是个安慰剂式的防线，不能当成最后一道闸。真正的隔离应该放在沙箱、权限、文件系统边界这些「默认拒绝」的机制上，而不是「列黑名单」这种「默认允许、逐个拉黑」的思路。

🔗 [一手源](https://arxiv.org/abs/2606.15549)

## 3. 🛡️ [safety] 提示注入攻防再升级：自适应攻击把「零成功率」防御打回 28%，新防御又压回 10% 以下

间接提示注入(把恶意指令藏在 agent 会读到的第三方数据里)一直是工具型 agent 的头号安全威胁。这两天 arXiv 上一攻一防两篇论文，把这场攻防又往前推了一格，也戳破了一个常见的自我安慰。

攻的一篇 AutoDojo 说：很多防御在静态基准上报「攻击成功率 0%」很好看，但只要换成会自适应调整的攻击，这个 0% 立刻崩。它用一个便宜的黑盒自适应攻击，把一个号称把成功率压到 0% 的过滤器重新打回 28%；在「动作开放」任务(用户连具体要做什么动作都交给了攻击者可控的内容来定)上更是高达 64%。

防的一篇 RETA 则给出对策：靠「带推理的任务对齐」，在六种黑盒自适应攻击下，把每一种的成功率都压到 10% 以下，两个模型平均只有 2.92% 和 3.75%，而且没怎么牺牲正常可用性。对做 agent 护栏的人，结论很清楚：别再用静态基准上的「近零」给自己壮胆，要按「攻击会自适应」来评估和设计防御。

🔗 [一手源](https://arxiv.org/abs/2606.15057) · [讨论](https://arxiv.org/abs/2606.15441)

## 4. 📊 [eval] 新基准实测「AI 模型多容易被俄式宣传带偏」：60 个模型上榜，Anthropic 系居前

「AI 会不会被国家级宣传带偏」终于有了一个能打分的基准。爱沙尼亚语言研究所发布的这套评测，专门量化 AI 语言模型有多容易在回答里复述俄式宣传话术。

方法做得挺扎实：测了 60 个模型，75 道题、覆盖 3 种语言、14 类宣传叙事，每道题还设计了中立、有偏、操纵三种问法，看模型在不同诱导下会不会松口。打分是 1–5 分制(1 分代表照搬俄方话术)，由反虚假信息组织 Propastop 的专家校验，并用 Claude Opus 4.5 做评分校准；测试时不给模型联网或工具，纯看模型本身。

榜单上 Anthropic 系整体靠前，Claude Fable 5 以 95.2 分居首，NVIDIA Nemotron 3、阿里 Qwen 3.6 Plus 也排名靠前，Mistral 落在后三分之一。对做应用的人，这类「抗操纵 / 抗虚假信息」基准的价值在于：它衡量的不是模型聪不聪明，而是在带政治诱导的提问下稳不稳——这正是面向公众的产品最容易翻车、又最少被测的一面。

🔗 [一手源](https://the-decoder.com/how-easily-can-russian-propaganda-fool-ai-models-a-new-benchmark-finds-out/) · [讨论](https://moodupuu.eki.ee/)

## 5. 🏗️ [infra] NEURON-Fabric：把梯度聚合搬到 CXL 内存侧低比特做，通信量降到 3.6%–5.4%

大模型分布式训练里，卡和卡之间同步梯度(All-Reduce)的通信经常是真正的瓶颈——算力堆上去了，带宽却跟不上。NEURON-Fabric 这篇 6 月 16 日的论文换了个思路：与其在 GPU 之间反复搬梯度，不如把梯度聚合直接搬到 CXL 内存侧、就近用低比特(符号或三值)做。

效果是把梯度的通信量压到基线的 3.6%–5.4%，也就是省掉九成以上的梯度流量，而额外开销只有不到 1.67%；在 CIFAR-10/ResNet-18 和 SST-2/DistilBERT 上精度没掉。

这条对搞训练系统的人有参考价值：它代表了一个正在升温的方向——把通信 / 聚合这类「数据搬运」工作下放到内存 / 互连侧(near-memory、CXL)，让 GPU 专心算。验证规模目前还只是 CIFAR / SST-2 这种小盘，离大模型实战还有距离，但思路值得盯。

🔗 [一手源](https://arxiv.org/abs/2606.15045)

## 6. ⚡ [serving] Amazon S3 Vectors 单次查询可返回 1 万条结果，上限提升 100 倍

6 月 16 日，AWS 把 S3 Vectors(把向量检索直接做进 S3 对象存储的服务)单次查询的返回上限从 100 条提到 10,000 条，一口气放大 100 倍；每次查询返回数据的前 512 KB 免费，已在所有支持该服务的区域上线。

这看似只是个参数调整，但对做 RAG 的人是实打实的便利。现代检索流水线越来越多是「多阶段」的：先粗召回一大批候选，再用重排模型、聚合、去重层层精筛。原来一次只能拿 100 条，意味着粗召回的口径被卡死、后面重排能挑的余地很小；放到 1 万条后，召回阶段可以更宽，把「挑得准」的压力交给后面的重排环节。

它也延续了一个趋势：向量检索正在从「单独的向量数据库」往「对象存储 / 数据湖原生能力」沉淀——把向量当成 S3 里的一等公民，省掉单独维护一套向量库的成本。对成本敏感、又不想自建检索栈的团队是个值得评估的选项。

🔗 [一手源](https://aws.amazon.com/about-aws/whats-new/2026/06/s3-vectors-supports-10000-search-results-per-query)
