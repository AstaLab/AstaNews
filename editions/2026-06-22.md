## 1. 🧠 [model] GLM-5.2:开源模型首次可信追平 GPT-5.5 一档

智谱新发布的 GLM-5.2 做了一件以前开源模型没真正做到的事:同时在硬指标和老手实测里追上闭源前沿。它是 753B MoE、激活 40B,用 MIT 协议开源,支持 100 万上下文、单次能输出 13 万 token,在 PostTrainBench 上拿了 SOTA 还超过 GPT-5.5。

光看榜单容易夸张,所以更值得听的是人话:Jeremy Howard 上手用过之后说,它至少和 Opus 4.8、GPT-5.5 一样好。在综合的 AA-Briefcase 榜上它是 1266 分,没登顶(Fable 5 是 1587),但已经踏进了闭源大模型的同一档位。

对从业者最实在的是可得性。MIT 协议商用几乎无门槛,HuggingFace 提供免费推理,普通显卡也能用 llama.cpp 在本地跑量化版。一句话:前沿能力第一次可以不经过 API 账单和合规审查就拿到手。智谱旗下 Z.ai 甚至预测,2026 年底会出现开源的 Fable 级模型。

## 2. 💰 [business] 美国政府以国安为由,迫使 Anthropic 下架 Fable 5 与 Mythos 5

美国政府第一次对一款前沿 AI 模型直接按下了停止键。理由是国家安全,对象是 Anthropic 的 Fable 5 和 Mythos 5,起因则是 Amazon 一位研究员演示了绕过 Fable 5 护栏的方法。

问题在于,这种越狱几乎所有强模型都躲不掉。Anthropic 自己就回应说,同样的手法在别家模型上一样能复现。于是一批网络安全研究者联名发公开信,认为这种禁令本身才危险——它会吓退漏洞研究,中断社区对模型安全的公开审视,而真正的坏人并不会因为官方下架就拿不到模型。

更耐人寻味的是后续:多篇报道发现,禁令不但没打压 Anthropic,反而让它更出名了。与此同时,Nathan Lambert 撰文直言禁止开源 AI 是个错误。一边开源在逼近闭源,一边监管开始管发布权,这场"安全还是开放"的争论,被一款模型的下架推到了正中央。

## 3. 🛡️ [safety] GDM 发布 AI Control Roadmap v0.1,把网络安全框架搬进对齐

Google DeepMind 发布了第一版 AI Control Roadmap,这是头一家前沿实验室把"如何控制一个不可信的强模型"做成了系统化路线图,而不是零散论文。

它最聪明的地方是抄了网络安全的作业。借鉴 MITRE ATT&CK 那套成熟的"对手战术编目",GDM 提出了叫 TRAIT&R 的对抗分类法,把模型可能搞的"小动作"分门别类,再对症下药。威胁分三类:失控、暗中搞破坏、直接危害;防御则拆成可分级的台阶——检测从 D1 到 D4,响应从 R1 到 R3,一共 15 项具体措施,从读思维链一路到监控模型内部激活、再到关停基础设施。

对行业最大的价值是可操作:以前做控制不知从何下手,现在有了一份能逐条对照、还能看出自己处在哪一级的清单。版本号停在 v0.1,作者也坦诚框架还没经过实战检验,但作为一个公开、可被批评和扩展的起点,它给所有人提供了一个能对标的蓝图。

## 4. 🛡️ [safety] DiffusionGemma 透明度审计:文本扩散模型的可解释性首测

文本扩散模型被看作自回归的接班人之一,但有个担忧一直悬着:它一口气并行生成整段文字,而不是一个词一个词往外蹦,那还能看懂它在想什么吗?GDM 的可解释性团队和扩散团队联手,做了第一份系统回答。

第一个数字很唬人:DiffusionGemma 一开始的"不透明程度",是同款自回归 Gemma 的 28.6 倍,像是黑箱了一个数量级。但关键在第二个数字——把现成的可解释性方法适配上去之后,差距瞬间缩到只有 1.1 倍。也就是说,它一开始显得难懂,很大程度是因为工具用错了地方,方法对路之后,两者透明度几乎打平。结论偏乐观:扩散这条路在可解释性上不会比自回归差太多。

过程中还冒出两个扩散独有的现象:模型会回头修改自己之前"写定"的内容(回溯自校正),推理也不必按文字先后顺序展开(非时序推理)。这提醒大家,给扩散模型做解释不能照抄自回归那套。完整结果见论文 arXiv:2606.20560。

## 5. 🏗️ [infra] AI 算力的真瓶颈是电力:已签 1.3 GW,四年还差 4.7 GW

聊 AI 算力,大家习惯盯着 GPU:谁囤了多少卡、什么制程。但 a16z 的 Anjney Midha 用算力电网运营商 AMP 的数据提醒大家,真正的瓶颈在电。

数据很直白:AMP 管线里已经签约 1.3 GW 电力,而未来四年的需求是 6 GW,中间差着 4.7 GW——相当于好几座顶级数据中心的用电凭空缺口。更吓人的是节奏:此前不少人预期 2026 年底算力会过剩,结果这个预期在六周内就蒸发了。

Midha 的判断是,这是结构性短缺,不是暂时的供需错配。区别很关键:错配能靠多买设备、产能爬坡自己缓解;结构性短缺卡在电网建设、输电许可、并网周期这些以"年"计、砸钱也快不了的环节上。对正在规划集群扩容的团队,这意味着卡脖子的可能不是买不到卡,而是接不到电——尤其对家底不如超大规模厂商的独立实验室,未来四年算力受限几乎是定局。

## 6. 🔧 [devtool] Qwen Code:阿里开源终端 coding agent,25K star、全协议不绑模型商

阿里开源的终端 coding agent Qwen Code 已经攒到 25.4K 个 GitHub star,它最大的卖点就俩字:不绑。

所谓不绑,是说它同时支持 OpenAI、Anthropic、Gemini、Qwen、Ollama 多种协议。你可以贵活儿用前沿闭源模型、跑量的活儿切到本地 Ollama 或自家 Qwen,上层的工作流一行不用改。在模型天天更新、价格频繁变动的当下,不被某一家锁死本身就是种保险。

功能也齐:跨会话记忆(auto-memory)、子 agent、MCP 全都有;入口更是从终端铺到 VS Code / JetBrains / Zed 插件、桌面应用、后台 daemon,甚至做了钉钉、飞书、微信、Telegram 的机器人——国内团队能直接在 IM 里喊它干活。项目用 TypeScript 写,累计 6515 次提交、498 个版本,最新到 v0.18.5,维护相当活跃。一句话:想自建 coding 工作流又不想押注单一模型商,这是套现成骨架。
